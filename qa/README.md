# QA — тест-планы BlockDuel 9×9

Полномасштабные матрицы ручных проверок по платформам. Файлы **генерируются**,
вручную не редактируются.

## Файлы

| Файл | Платформа | Пунктов |
|------|-----------|---------|
| `TEST_PLAN_web.md` | Web (Chrome/Safari/Firefox/Edge, desktop+mobile) | ~2360 |
| `TEST_PLAN_macos.md` | macOS desktop | ~2360 |
| `TEST_PLAN_ios.md` | iOS (iPhone) | ~2350 |
| `TEST_PLAN_ipados.md` | iPadOS (планшет) | ~2370 |

Всего ~9.4 тыс. пунктов.

## Как устроено

Матрица собирается из измерений предметной области (экраны × темы × состояния;
фигуры × ориентации × постановка; режимы × правила; онлайн-флоу × сетевые
условия; протокол; магазин; **реальные 120 достижений × 12 фасетов**; настройки;
звук; хранение/sync/auth; локализация; производительность; ошибки) плюс
платформо-специфичные разделы.

Достижения подтягиваются из `flutter/lib/achievements/definitions.dart` — при
изменении каталога перегенерируй планы.

## Регенерация

```bash
python3 tools/gen_test_plan.py
```

Перезаписывает 4 файла в `qa/`. Генератор детерминированный (одинаковый ввод →
одинаковый вывод), поэтому диффы читаемы.

## Как пользоваться

- **Smoke (каждая сборка):** прогоняй только пункты с маркером `[C]` —
  критический срез (~14% матрицы).
- **Полный регресс (перед релизом/cut-over):** весь файл платформы.
- Чекбоксы `- [ ]` отмечай прямо в копии файла под конкретный прогон/билд.

## Связанное

- Автотесты (Vitest/Flutter `flutter test`) покрывают ядро/нотифайеры/виджеты —
  это нижний слой; ручные планы покрывают то, что автотест не ловит (рендер,
  платформенная интеграция, ввод, звук, сеть в реальных условиях).
- Парность интерфейса с TS — `INTERFACE_PARITY.md` в корне.
