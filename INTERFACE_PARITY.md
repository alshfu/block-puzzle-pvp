# INTERFACE_PARITY — паритет UI Flutter ↔ TS (web-first)

> Сравнение интерфейса Flutter-порта с эталонной TS/React-версией (она всех
> устраивала, «всё работало»). Источник правды по UI — TS-экраны `legacy-ts/ui/`
> (бывший `src/ui/`). Цель — довести Flutter-web до уровня TS. Отмечаем `[x]`
> закрытое, `[~]` в работе, `[ ]` пробел. Обновлено: 2026-06-10.
>
> **Статус: паритет практически достигнут.** После Фазы 3b (онлайн-статистика →
> PvP-ачивки + разбивка очков в ResultOverlay) функциональных регрессов
> относительно TS не осталось. Открыто лишь несколько мелочей (язык только RU,
> экспорт/импорт прогресса, инлайн-рука соперника, кнопка «Отразить») и
> визуальный pixel-parity gate (ручная приёмка пользователя).

## ✅ Уже на паритете
Меню (hero/режимы/resume/темы/FloatingTheme), игровой экран (доска/рука/таймер/
скорборд/контролы), пауза, конфетти, combo-flash, тосты, **онлайн целиком**
(лобби, матч, reconnect, ремач, ELO-лидерборд), auth (Google web) + Firestore
sync, 3 темы, звук+музыка, маскоты/фон.

## ⚠️ Функциональные регрессы (было в TS, в Flutter урезано)
- [x] **SetupScreen перед матчем** (2026-06-08) — выбор уровня бота (easy/medium/
      hard), правил (повороты/отражения/размер руки), таймера (хардкор/норма/
      казуал). `/setup/:mode` → конфиг в query → `/game`. botvbot пока один
      уровень на обоих ботов (A/B раздельно — позже).
- [x] **Настройки до паритета** (2026-06-08) — громкости звука/музыки (слайдеры,
      проброшены в плееры), вибрация (off/light/strong, HapticFeedback), тогглы
      анимаций (конфетти/маскоты/ghost/reduceMotion — проброшены), задержка бота
      (слайдер → game_notifier), правила/бот/блиц по умолчанию (→ SetupScreen),
      сброс прогресса (2 шага). *Отложено:* экспорт/импорт JSON (web-загрузка
      файла), скорость анимации очистки (очистка пока мгновенная).
- [x] **Ачивки** — каталог **120** в 5 категориях (`definitions.dart`) + движок
      прогресса (`engine.dart`) + `Stats`/`StatsController` + экран с категориями/
      прогресс-барами/скрытыми/+XP (2026-06-08). Оффлайн-ачивки работают;
      **онлайн-статистика (3b) закрыта (2026-06-09):** PvP-ачивки наполняются из
      онлайн-матчей (ELO через `myElo`, opponents/themes/gaps трекаются), тосты,
      экран `/stats`.
- [x] **ResultOverlay** — маскот, счёт, +XP/+монеты, список новых ачивок,
      баннер серии побед (×N), реванш/меню + **детальная разбивка очков
      (база/комбо/perfect) (2026-06-09)**.
- [x] **Профиль** (2026-06-08) — 7 стат-плиток (партии/победы/винрейт/рекорд/
      мульти/линии/серия+лучшая) + **12 аватаров** (паритет с TS) + ссылка на
      ачивки + сброс статы.

## ❌ Отсутствует целиком
- [x] **Магазин** (ShopScreen): вкладки power-ups + скины, две валюты (2026-06-08,
      Фаза 7.1) — `/shop`, кошелёк, покупка/надевание, персист, 6 тестов.
- [x] **Power-ups** (6: подсказка/обмен руки/строка/столбец/бомба/умный ход)
      (2026-06-08, Фаза 7.3) — панель в игре + эффекты в GameNotifier + расход
      из инвентаря; режим выбора клетки для палочки/бомбы, подсветка подсказки.
- [x] **Скины клеток** (6) + **кристаллы** (премиум-валюта) (2026-06-08, Фаза 7.1–7.2) —
      покупка/надевание + применение к рендеру доски (gem/candy/bullet/neon/pixel на Canvas).
- [x] **Tutorial** (5 шагов, +50 монет) (2026-06-08) — `/tutorial`: поставь/закрой
      строку/поверни+столбец/закрой бокс/комбо; прогресс-бар, статус, «Заново»,
      награда один раз (флаг bd_tutorial_done). Переиспользует BoardView/HandView.
- [x] **Arcade** (соло на рекорд) (2026-06-08) — MatchMode.arcade: ходит только
      игрок 0, ход не передаётся, партия до собственного тупика; solo-скорборд и
      экран результата «Игра окончена / счёт N».
- [x] **Вибрация** (haptics) — off/light/strong через `HapticFeedback` (в настройках).
- [ ] **Экспорт/импорт прогресса** (web-загрузка/выгрузка JSON).
- [x] **Фигура в руке в текущей ориентации + поворот тапом** (2026-06-08) —
      HandView рисует выбранную фигуру в её ориентации; повторный тап = поворот.
- [x] **Пилот в онлайн-матче** (2026-06-10) — паритет с TS `PilotMode = "online"`:
      `OnlineGameNotifier.pilotPlayTurn()` (ход через ядро + `placeAt`, anti-cheat
      по построению) + общий `PilotHud` на OnlineGameScreen (dev-гейт). E2E-обвязка:
      `tools/online-bot-client.ts` — headless-оппонент (lobby→room, играет ядром,
      MAX_TURNS→resign), проверено на macOS-сборке против локального сервера.
- [ ] Инлайн-рука соперника в скорборде; отдельная кнопка **«⇄ Отразить»**.
