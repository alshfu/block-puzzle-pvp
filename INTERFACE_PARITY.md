# INTERFACE_PARITY — паритет UI Flutter ↔ TS (web-first)

> Сравнение интерфейса Flutter-порта с эталонной TS/React-версией (она всех
> устраивала, «всё работало»). Источник правды по UI — TS-экраны `src/ui/`.
> Цель — довести Flutter-web до уровня TS. Отмечаем `[x]` закрытое, `[~]` в
> работе, `[ ]` пробел. Обновлено: 2026-06-08.

## ✅ Уже на паритете
Меню (hero/режимы/resume/темы/FloatingTheme), игровой экран (доска/рука/таймер/
скорборд/контролы), пауза, конфетти, combo-flash, тосты, **онлайн целиком**
(лобби, матч, reconnect, ремач, ELO-лидерборд), auth (Google web) + Firestore
sync, 3 темы, звук+музыка, маскоты/фон.

## ⚠️ Функциональные регрессы (было в TS, в Flutter урезано)
- [x] **SetupScreen перед матчем** (2026-06-08) — выбор уровня бота (easy/medium/
      hard), правил (повороты/отражения/размер руки), таймера (хардкор/норма/
      казуал). `/setup/:mode` → конфиг в query → `/game`. botvbot пока один
      уровень на обоих ботов (A/B раздельно — позже).
- [x] **Настройки до паритета** (2026-06-08) — громкости звука/музыки (слайдеры,
      проброшены в плееры), вибрация (off/light/strong, HapticFeedback), тогглы
      анимаций (конфетти/маскоты/ghost/reduceMotion — проброшены), задержка бота
      (слайдер → game_notifier), правила/бот/блиц по умолчанию (→ SetupScreen),
      сброс прогресса (2 шага). *Отложено:* экспорт/импорт JSON (web-загрузка
      файла), скорость анимации очистки (очистка пока мгновенная).
- [ ] **Ачивки** — ~120 в 5 категориях (single/progressive/series/hidden/online)
      с прогресс-барами и +XP; сейчас 11 плоских lock/unlock.
- [ ] **ResultOverlay** — разбивка очков (база/комбо/perfect), XP-бар, баннер
      серии побед, реванш; сейчас простой оверлей (маскот+счёт+2 кнопки).
- [ ] **Профиль** — 12 стат-плиток + 12 аватаров + ссылка на ачивки + сброс
      статы; сейчас 4 статы / 8 аватаров.
