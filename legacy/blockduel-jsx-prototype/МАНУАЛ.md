# BlockDuel 9×9 — Мануал к прототипу

Интерактивный прототип PVP-головоломки «BlockDuel 9×9» с тремя переключаемыми темами оформления. Собран на React (без сборки — работает прямо в браузере через Babel).

---

## 1. Как запустить

1. Распакуйте архив целиком (важно сохранить структуру папки `blockduel/`).
2. Откройте файл **`BlockDuel 9x9.html`** в современном браузере (Chrome, Safari, Edge, Firefox).
   - Можно просто дважды кликнуть по файлу.
   - Если шрифты/скрипты не грузятся при открытии «с диска», запустите локальный сервер из папки:
     ```
     # python 3
     python -m http.server 8000
     # затем откройте http://localhost:8000/BlockDuel%209x9.html
     ```
3. Нужен интернет — подгружаются React и шрифты Google Fonts по CDN.

> Прототип рассчитан на **телефон в портретной ориентации** (на весь экран). На десктопе показывается колонка по центру шириной до 460px.

## 2. Структура файлов

```
blockduel/
├── BlockDuel 9x9.html   ← точка входа: весь CSS + подключение модулей
├── core.jsx             ← игровое ядро (чистая логика, без UI)
├── themes.jsx           ← 3 темы: токены, шрифты, декоративные фоны
├── components.jsx       ← UI-атомы и игровые компоненты (доска, рука, табло, таймер)
├── screens.jsx          ← экраны: меню, настройка матча, результат, переключатель тем
├── app.jsx              ← корень: навигация, состояние игры, игровой экран
└── МАНУАЛ.md            ← этот файл
```

Порядок загрузки модулей задан в конце `BlockDuel 9x9.html`:
`core → themes → components → screens → app`.
Каждый модуль экспортирует свои функции/компоненты в `window` (так они видны другим модулям при работе через Babel).

## 3. Три темы

Переключаются **на лету** в трёх местах:
- на главном меню (блок «ТЕМА ОФОРМЛЕНИЯ» внизу),
- в настройке матча (компактный ряд свотчей),
- в игре — парящей кнопкой в правом нижнем углу (циклически меняет тему).

Выбор темы сохраняется в `localStorage` (ключ `bd_theme`).

| Тема | Айди | Вайб |
|---|---|---|
| **Нейтральная** | `neutral` | Тёмная база из техдоки: янтарь (ты) / циан (соперник), радиальные подсветки. Тема по умолчанию. |
| **Магия & Карамель** | `candy` | Светлая пастель: небо-градиент, облака, радуга, падающие сердечки/звёздочки/блёстки, глянцевые «леденцовые» клетки. Шрифты Fredoka + Baloo 2. |
| **Ночной Страж** | `night` | Неон-нуар/готика: прожектор-луч с абстрактной эмблемой (ромб со звездой), силуэт ночного города, угловатые «комиксные» неоновые блоки. Шрифты Oswald + Share Tech Mono. |

> Все темы **оригинальные** — навеяны атмосферой, но не копируют существующие бренды (логотипы, персонажей и т.п.).

### Как темы устроены технически
В `themes.jsx` объект `THEMES` хранит для каждой темы:
- `vars` — набор CSS-переменных (цвета, радиусы, шрифты), которые навешиваются на корневой `.app-root` инлайн-стилем;
- `kind` — `"dark"` или `"light"` (влияет на контраст текста кнопок, оверлеи);
- `p0name` / `p1name` — как зовут игроков в этой теме (напр. «Пони-бот», «Тень»);
- декоративный фон рисует компонент `ThemeBackdrop` (свой для каждой темы).

Тематические визуальные нюансы (глянец клеток, неоновое свечение, скосы углов) лежат в `BlockDuel 9x9.html` в CSS-селекторах вида `[data-theme="candy"] …` / `[data-theme="night"] …`.

## 4. Карта экранов

```
Меню ──▶ Играть ──┬─ С ботом ──┐
                  └─ Вдвоём ────┴─▶ Настройка матча ──▶ Игра ──▶ Результат
                                                          ▲          │
                                                          └─ Реванш ──┘
                                                          В меню ◀────┘
   (Онлайн — задизейблен, «скоро»)
```

### 4.1. Меню
Логотип, кнопка-герой «Играть» (раскрывает выбор режима), вторичные кнопки (Профиль/Ачивки/Настройки — заглушки), аватар и уровень игрока, переключатель тем.

### 4.2. Настройка матча
- **Соперник** (для режима с ботом): Тупой / Умный / Сложный.
- **Правила**: тумблеры «Повороты» и «Отражения», размер руки (2/3/4).
- **Блиц** (время на ход): Хардкор 8→2с · Норма 12→3с · Казуал 20→6с.

### 4.3. Игровой экран
Сверху вниз: шапка (лого, бейдж режима, пауза) → табло с кольцом-таймером → рука соперника (наблюдение) → доска 9×9 → строка статуса → твоя рука → контролы трансформации.

### 4.4. Результат
Модалка: Победа / Поражение / Ничья, счёт A:B, +XP с полосой, кнопки «Реванш» и «В меню».

## 5. Правила и управление

### Ход
1. Выбери фигуру в своей руке (тап / клик).
2. (Опц.) Поверни / отрази её.
3. Наведи на поле — появится «призрак»: **зелёный** — встанет, **красный** — нельзя.
4. Кликни по клетке, чтобы поставить.

### Очистка и очки
- Заполнил целиком **строку**, **столбец** или **квадрат 3×3** — они очищаются.
- Базовые очки растут с числом линий за один ход (1 → 3 → 6 → 10 …).
- **Комбо**: серия ходов с очистками повышает множитель.
- **Perfect clear** (доска опустела) — крупный бонус.
- Над местом постановки всплывает «+N».

### Таймер
У активного игрока крутится кольцо. За ~3 секунды до нуля оно мигает красным. На нуле — **авто-ход** случайной валидной фигурой (не пропуск). С раундами время сокращается до нижнего предела.

### Конец партии
Когда активному игроку **некуда ставить** ни одну фигуру из руки — партия завершается, побеждает тот, у кого больше очков.

### Горячие клавиши (десктоп)
| Клавиша | Действие |
|---|---|
| `R` (или `К`) | Повернуть выбранную фигуру |
| `F` (или `А`) | Отразить |
| `Esc` | Снять выбор |
| Наведение мышью | Призрак-предпросмотр |

## 6. Режимы

- **С ботом** — играешь против ИИ. Уровни: Тупой (случайный валидный ход), Умный (жадная эвристика с небольшой случайностью), Сложный (лучший ход по эвристике). Рука бота видна для просмотра («наблюдай»).
- **Вдвоём (hot-seat)** — два человека ходят по очереди на одном устройстве на общей доске. Активная рука внизу подсвечена, подпись «Ходит: Игрок N».

## 7. Игровое ядро (`core.jsx`)

Чистая, детерминированная логика без обращений к UI/DOM. Ключевое:

| Функция | Назначение |
|---|---|
| `makeRng(seed)` | Детерминированный PRNG (mulberry32) |
| `makeBag(seed)` | 7-bag — выдаёт фигуры |
| `emptyBoard()` / `cloneBoard()` | Создать / скопировать поле 9×9 |
| `canPlace(board, cells, r, c)` | Можно ли поставить |
| `place(...)` | Поставить (мутирует) |
| `findClears(board)` | Найти полные строки/столбцы/боксы |
| `applyClears(...)` | Очистить клетки |
| `scoreForMove(N, combo, perfect, cfg)` | Очки за ход |
| `enumerateMoves` / `hasAnyMove` / `pieceHasMove` | Перебор ходов, проверка тупика |
| `chooseBotMove(board, hand, level, cfg, rng)` | Ход бота |

Фигуры: 7 классических тетромино — `I O T S Z J L`.

> ⚠️ Бот в прототипе **упрощён** (жадная одношаговая эвристика). Для реальной разницы уровней по ТЗ нужны массовые bot-vs-bot симуляции и более глубокий перебор.

## 8. Что можно докрутить (не реализовано в прототипе)

- Экраны **Профиль / Ачивки / Настройки** — сейчас кнопки-заглушки.
- В hot-seat — нейтральная заставка **«Передай устройство»** между ходами.
- **Звук и гаптика** (в ТЗ есть маппинг событий).
- **Онлайн** (фаза 3): матчмейкинг, лобби, реконнект.
- Реальный **профиль/статистика/XP** с сохранением и прогрессом ачивок.
- Полноценный **режим дальтоника** (паттерны на фигурах).

## 9. Как добавить свою тему

1. В `themes.jsx` добавьте запись в `THEMES` (скопируйте `neutral` как основу) и айди в массив `THEME_ORDER`.
2. Пропишите палитру в `vars` (цвета `--p0/--p1`, фоны, радиусы, шрифты).
3. При желании нарисуйте декоративный фон — добавьте ветку в `ThemeBackdrop`.
4. Тематические нюансы клеток/кнопок добавьте в `BlockDuel 9x9.html` селекторами `[data-theme="ваш-айди"] …`.
5. Подключите новые шрифты в `<link>` Google Fonts в шапке HTML.
